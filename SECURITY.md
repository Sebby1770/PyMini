# Security Policy

The unreleased main branch is the currently supported development version.

Please report vulnerabilities privately through the repository's GitHub **Security** tab by
opening a private vulnerability report for `Sebby1770/PyMini`. Do not publish interpreter
escape techniques or denial-of-service payloads in a public issue. Include the parser/engine,
source payload, observed impact, execution limits, and affected commit.

PyMini is an educational interpreter and is not designed to execute untrusted code.
